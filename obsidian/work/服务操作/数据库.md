# 数据库

## 方案一：存储过程

```text
推荐，自动清所有表。
在 Navicat 里整段选中执行，包含 DELIMITER。
```

```sql
USE `kingdom_game`;
DROP PROCEDURE IF EXISTS `truncate_all_tables`;
DELIMITER $$
CREATE PROCEDURE `truncate_all_tables`()
BEGIN
    DECLARE done INT DEFAULT 0;
    DECLARE tbl VARCHAR(255);
    DECLARE cur CURSOR FOR
        SELECT table_name
        FROM information_schema.tables
        WHERE table_schema = 'kingdom_game'
          AND table_type = 'BASE TABLE';
    DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = 1;
    SET FOREIGN_KEY_CHECKS = 0;
    OPEN cur;
    read_loop: LOOP
        FETCH cur INTO tbl;
        IF done = 1 THEN
            LEAVE read_loop;
        END IF;
        SET @s = CONCAT('TRUNCATE TABLE `', tbl, '`');
        PREPARE stmt FROM @s;
        EXECUTE stmt;
        DEALLOCATE PREPARE stmt;
    END LOOP;
    CLOSE cur;
    SET FOREIGN_KEY_CHECKS = 1;
END$$
DELIMITER ;
CALL `truncate_all_tables`();
DROP PROCEDURE IF EXISTS `truncate_all_tables`;
```

## 方案二：静态脚本

```text
最稳，不依赖存储过程。
如果 Navicat 对 DELIMITER 支持不好，直接执行这段。
```

```sql
USE `kingdom_game`;
SET FOREIGN_KEY_CHECKS = 0;
TRUNCATE TABLE `ranking_info`;
TRUNCATE TABLE `role`;
TRUNCATE TABLE `role_ark`;
TRUNCATE TABLE `role_attribute`;
TRUNCATE TABLE `role_branch_task`;
TRUNCATE TABLE `role_buff`;
TRUNCATE TABLE `role_daily`;
TRUNCATE TABLE `role_daily_task`;
TRUNCATE TABLE `role_daily_task_point`;
TRUNCATE TABLE `role_data`;
TRUNCATE TABLE `role_equipment`;
TRUNCATE TABLE `role_item`;
TRUNCATE TABLE `role_main_task`;
TRUNCATE TABLE `role_main_task_progress`;
TRUNCATE TABLE `role_period_task`;
TRUNCATE TABLE `role_pub`;
TRUNCATE TABLE `role_ship`;
TRUNCATE TABLE `role_shop`;
TRUNCATE TABLE `role_timed_attribute`;
TRUNCATE TABLE `role_troops`;
TRUNCATE TABLE `side_army`;
TRUNCATE TABLE `side_level_chapter_common_data`;
TRUNCATE TABLE `side_level_info`;
TRUNCATE TABLE `side_level_room`;
TRUNCATE TABLE `side_role`;
SET FOREIGN_KEY_CHECKS = 1;
```

## 备注

- 之前给的动态 `PREPARE` 方案不适用。
- 逐条执行时，先跑 `SET FOREIGN_KEY_CHECKS = 0`，最后跑 `SET FOREIGN_KEY_CHECKS = 1`。
